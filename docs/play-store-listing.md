## 3. 데이터 보안 (Data safety) 양식 답변

> 개인정보처리방침과 1:1 정합. Play Console "앱 콘텐츠 > 데이터 보안"에 그대로 입력.

**개요**
- 데이터 수집/공유? **수집 예 / (광고용)공유 아니오**
- 전송 중 암호화? **예**(전 통신 HTTPS)
- 사용자가 데이터 삭제 요청 가능? **예** (정산=내역 탭 직접 삭제 / 계정=요청 — §6 참조)

**수집 데이터 항목** (모두 "수집함 · 공유 안 함 · 앱 기능/계정관리 목적 · 암호화됨")

| Play 분류 | 항목 | 수집 | 선택/필수 | 목적 |
|---|---|---|---|---|
| 개인 정보 | 이름/닉네임 | 예 | 로그인 시 필수 | 앱 기능·계정 관리 |
| 개인 정보 | 이메일 주소 | 예 | 로그인 시 필수 | 계정 관리·인증 |
| 사진 | 프로필 이미지(소셜) | 예 | 선택 | 계정 표시 |
| 금융 정보 | 기타 금융 정보(받을 계좌 은행/번호/예금주) | 예 | 선택(사용자 입력) | 앱 기능(송금 안내) |
| 앱 활동 | 사용자가 만든 콘텐츠(정산 제목·금액·참여자 이름·날짜) | 예 | 필수 | 앱 기능 |

**IP 주소:** 요청 제한(악용 방지)용으로 **일시적으로만** 사용·만료 → Google "보안/사기 방지 목적 일시 처리" 예외에 해당해 별도 수집 항목 미표기 가능. (보수적으로 표기하려면 "기기/기타 ID" 아님 — 해당 분류 없음. 표기 시 "앱 활동 외 진단" 부적합 → 미표기 권장.)

**공유(3자 제공):** 없음. 인프라 처리자(Supabase·Vercel·Upstash·Google·Kakao)는 "처리 위탁"이지 데이터 보안상 '공유' 아님(기능 제공 한정).

## 6. 출시 전 확인/보완

1. ✅ **계정 삭제 경로 구현됨** (Play 정책 필수) — 마이 탭 하단 "계정 삭제"(2단계 확인). `deleteMyAccountAction` → auth 유저 삭제(이메일·이름·저장계좌·내 모임 제거, 공유 정산은 owner null로 비식별화). 데이터 보안 양식의 "데이터 삭제 요청" = **인앱 삭제 있음**으로 응답(+보조로 `payven.kr/privacy` 이메일). build/lint/test green. ⏳ **로그인 폰 스모크 잔여**(실제로 삭제되니 버리는 계정으로 테스트).
2. ✅ **피처 그래픽** — `store-assets/feature-graphic.html`을 브라우저로 열고 'PNG로 저장' 클릭 → `payven-feature-graphic-1024x500.png` 받기. 글자/색은 그 파일의 SVG 텍스트만 고치면 됨.
3. **폰 스크린샷 2~8장** (레시피) — 폰에서 `payven.kr` 접속 → 다음 5컷 캡처(**라이트 모드 권장**, 9:16, 상단 상태바 깔끔하게):
   ① 홈(금액 입력 화면) ② 항목별/메뉴별 입력 ③ 정산 결과(보낼 금액·토스 버튼) ④ 공유 링크 화면(친구가 보는 자기 몫) ⑤ 내역 또는 내 모임. (Play는 첫 2~3컷만 노출되니 ①②③ 순서가 중요.)
4. 첫 출시 국가 = **대한민국**(이후 확장).

## 7. 번호 받은 뒤 실행 순서 (요약)
1. D&B에서 회사명·주소 = 사업자등록증 일치 확인(3중 일치)
2. Play Console → Organization 계정 생성($25, 새 결제 프로필, 웹사이트 payven.kr)
3. 앱 만들기 → 이 문서로 스토어 등록정보·데이터보안·콘텐츠등급 입력
4. **AAB 업로드**(`페이븐.aab`) → 프로덕션(또는 비공개 테스트)
5. Play Console → App integrity → **App signing key SHA-256 복사** → 나에게 전달
6. (나) assetlinks에 **2번째 지문** 추가·재배포
7. 테스트 기기 설치 → **주소창 사라짐 확인** → 프로덕션 제출 🚀
