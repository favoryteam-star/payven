## 성장 엔진 = 링크 루프
검색·네트워크·돈이 없으니 유일한 무료 확산 장치:
> 정산 1건 → 가입 안 한 참여자 N명에 링크 노출 → 일부가 *자기* 정산 생성 → 또 N명…

```
K = (정산당 노출 인원) × (그중 새 생성자 전환율)
예) 4명 모임 → 3명 노출 × 10% = K 0.3
```
- K≥1 = 자가증식(정산앱에선 드묾, 기대 금지)
- K 0.3~0.7 = 씨앗을 1.4~3배 증폭(엔진 아닌 **증폭기**)
- K<0.1 = 루프 죽음
→ **성장의 8할 = K를 0→0.3+로 끌어올리기.**

## 🔑 1순위 레버 = 로그인 게이트
만들기 = 로그인 필요(보기·공유는 무로그인). 링크 받은 사람이 "나도 만들래" 해도 *로그인부터* 막혀 **루프의 목을 조름.**
- 방향: **분담 다 짜고 결과까지 보여준 뒤 → 저장/공유(링크 생성)에서만 로그인.** 가치 먼저, 게이트 나중.
- 기술: M4 전엔 무로그인 생성이었음 → owner_id null 허용으로 되돌릴 수 있음(withRateLimit은 유지).
- **데이터로 확인 후 결정**(아래 측정의 `login_gate_shown` 비율).

## 랜딩 결정 = tool-first (별도 마케팅 랜딩 ❌)
도구 페이지 자체가 곧 랜딩(payven은 이미 홈=만들기 도구). 우리 트래픽(광고·링크 루프)은 둘 다 "지금 바로 쓰고 싶은" 사람이라 인터스티셜 한 장이 곧 이탈.
- 도구 위에 **2초 가치 한 줄**(예: "복잡한 정산도 1분 · 무가입 링크 · 영수증 찍으면 자동입력")로 설명을 도구 안에 박기.
- 광고 랜딩도 **도구로 직행**(utm) — 마케팅 페이지 금지.
- 예외(Phase 2): SEO/ASO용 **콘텐츠 페이지**는 발견용으로 가치 있음(전환 게이트는 아님).

## 채널별 정직한 기대치
| 채널 | 역할 | 현실 |
|---|---|---|
| 링크 루프 | 🔑 주 엔진 | 유일한 공짜 확산. K 올리기가 전부. K<1이라 증폭기 |
| 안드로이드 출시 | 발견성·습관·신뢰 | Play 검색(ASO)·아이콘 재방문. 폭발 아닌 토대 |
| 소액 광고 | 씨앗 주입 | 경쟁0이라 쌈. 무료앱이라 지속 광고 정당화 어려움 → 검증·시딩용만 |
| SEO/콘텐츠 | 보조 | 정보성 더치페이 트래픽은 있으나 도구 전환 약함. 느림·후순위 |
| 커뮤니티 | 시딩 한 줌 | 네트워크 없어 제한적. 메이커 커뮤니티(디스콰이엇/긱뉴스)는 헛신호 위험 → 후순위 |

## 단계별 계획 (게이트식 — 통과해야 다음)
**Phase 0 — 검증(지금~3주, 진행 중):** 목표 = 유저 수 아닌 *학습*. 활성화율 + 루프(K) 측정.
- 행동: 실제 1건(트랙 A·공짜·최고 신호) + ₩3~5만 검색광고(콜드 ~수백 클릭).
- 🚦 게이트: 활성화 ≥5~10% **그리고** viewer→creator 신호 → Phase 1. 둘 다 미지근 → **취미 유틸 유지(이미 라이브)·성장 투자 중단.**

**Phase 1 — 첫 100 실유저(검증되면, 1~2개월):** 안드 출시(ASO) + 루프 가동 + 로그인 게이트 개선(K↑) + 상시 소액 시딩. 타깃 = 재방문 생성자 ~100명·K 0.3+ 굳히기.

**Phase 2 — 확대(루프 살아있으면):** 정보성 더치페이 SEO/콘텐츠, 추천 넛지, iOS, CAC<가치일 때만 광고. 타깃 = 성장률·retention.

## 측정 (Phase 0 — 2026-06-26 구현, 코드)
Vercel Web Analytics(쿠키리스) + 커스텀 퍼널 이벤트(`lib/analytics.ts`):
```
방문(자동) → create_attempted → ( login_gate_shown | settlement_created )
```
⚠️ **플랜 현실(Hobby, 2026-06-26 확인):** 커스텀 이벤트 3종은 **Vercel Pro 전용** → Hobby 대시보드 Events 패널엔 안 보임. **페이지뷰·Referrers·UTM Parameters는 Hobby에서 됨.** 그래서 Phase 0은 **Pro 없이** 아래 조합으로 측정한다(이벤트 코드는 유지 — Pro 켜는 순간 즉시 작동):
- **활성화율 = Supabase 생성 수(분자) ÷ Vercel UTM 방문(분모).** 광고를 `payven.kr/?utm_source=naver`로 직행 → Vercel **UTM Parameters** 탭에서 그 source 방문 수.
- **분자(정산 생성)**: Supabase `groups`(created_at·owner_id) 측정 창 기준 직접 카운트.
- **로그인 게이트 이탈**: Hobby 직접 측정 불가 → **방문−생성 갭으로 추정**(대부분 게이트). 정밀 숫자 필요 시 한 달 Pro.
- viewer→creator는 트랙 A 실제 1건에서 눈으로 + DB.
- 켜기: Vercel → payven → Analytics → Enable(페이지뷰·UTM 무료).
